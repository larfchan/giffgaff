# giffgaff 保号方案

一个手动触发的极简 Android 工具。点击一次后，应用会申请系统首选的**非 VPN 蜂窝网络**，向默认目标 `8.8.8.8:53` 发送一个 **17 字节的合法 DNS 查询**。

## 发送内容

- 默认公网 IP：Google Public DNS `8.8.8.8`
- 应用负载：17 字节的 DNS 根域 A 记录查询
- 接收：应用不调用 `receive()`，但 DNS 服务器仍可能回复，系统也可能产生少量网络探测

## 注意

1. 请在设置中确保漫游只给本 App 权限，且手动设定系统级流量上限更为保险。
2. 请关闭智能切卡。
3. 请关闭无关 VPN 软件。
4. 发送之后，请等待一段时间自行查看是否有余额变动，以免保号失败。提前几天操作，不要等到六个月期限当天才操作。如果失败，请转用发送短信方式。

公网 IP 可以修改，但只接受数字公网 IPv4/IPv6；由于端口和内容固定为 DNS/UDP 53，通常保持默认的 `8.8.8.8` 即可。

## 保号与费用边界

giffgaff 当前帮助页说明，号码至少每六个月需要进行一次余额变动，使用移动数据连接互联网属于消费活动之一。截至本项目发布 （2026-07-17） ，[giffgaff 中国漫游价格页](https://www.giffgaff.com/roaming/china)显示数据为 20p/MB、发送短信为 30p/条。公开页面没有说明最小计费单位、最低扣款或取整方式，所以不能保证一次点击发送的实际扣款的具体数值。
